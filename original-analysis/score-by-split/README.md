This directory computes aggregate verifier score and score rate for split groups
(top/rest) generated by `original-analysis/split`.

Score definition:

- `score` = sum of `verifier_result.rewards.reward` across trails in the group
- `score_rate` = `score / trail_count` for that group

It reports both normal and strict-499 split variants.

## Script

- `original-analysis/score-by-split/extract_run.py`

## Per-Run Extraction

Input run layout:

- `<run-dir>/meta/results.json`
- `<run-dir>/trials/<trial-id>/result.json`
- `<run-dir>/gateway-output/.../manifest.json` (or lifecycle fallback)
- split files under `<run-dir>/original-analysis/split/`:
  - `top-p-token-usage-two-groups.json`
  - `top-p-context-usage-two-groups.json`
  - `top-p-token-usage-two-groups.strict-499.json`
  - `top-p-context-usage-two-groups.strict-499.json`

Command:

```bash
python original-analysis/score-by-split/extract_run.py \
  --run-dir <run-dir>
```

Default output:

```text
<run-dir>/original-analysis/score-by-split/score-by-split-summary.json
```

Optional:

- `--output <json-path>` (only with `--run-dir`)

## Root Extraction

Batch mode:

```bash
python original-analysis/score-by-split/extract_run.py \
  --root-dir <root-dir>
```

This recursively discovers runs with split two-group files, writes per-run summaries,
and emits root CSV tables:

```text
<root-dir>/score-by-split-token-usage-two-group-table.csv
<root-dir>/score-by-split-context-usage-two-group-table.csv
<root-dir>/score-by-split-token-usage-two-group-table.strict-499.csv
<root-dir>/score-by-split-context-usage-two-group-table.strict-499.csv
```

Optional:

- `--max-procs <positive-int>`
- `--dry-run`

## Root CSV Columns

- `run_path`
- `selected_p`
- `group_top_trail_count`
- `group_top_score`
- `group_top_score_rate`
- `group_rest_trail_count`
- `group_rest_score`
- `group_rest_score_rate`
- `score_gap`
- `score_rate_gap`

Where:

- `score_gap = group_top_score - group_rest_score`
- `score_rate_gap = group_top_score_rate - group_rest_score_rate`
